## Design Context

### Users

ARTEMIS serves a human architect/operator working with Codex, Claude Code, and future coding agents. The user context is operational: deciding what should run, what is blocked, what needs evidence, and what is ready for human review.

### Brand Personality

Simple, state-of-the-art, complete. The interface should feel like an engineering control room: calm, precise, durable, and low-drama.

### Aesthetic Direction

Use a restrained operational UI, not a landing page. Prefer dense but readable information, strong typographic hierarchy, crisp dividers, and clear state. Avoid decorative gradients, generic cards, dark neon dashboards, and marketing hero sections.

### Design Principles

1. Show flow before detail.
2. Make state transitions visible.
3. Keep human decisions explicit.
4. Treat evidence as part of the work.
5. Prefer one useful surface over many clever abstractions.
